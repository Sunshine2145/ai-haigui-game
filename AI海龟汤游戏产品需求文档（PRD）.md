# 一、产品概述

## 1.1 产品定位

AI驱动的沉浸式悬疑海龟汤解谜游戏，以**单人AI互动推理**为核心玩法，依托大模型API实现智能应答，用户通过文字提问逐步推导故事真相，主打轻量、悬疑、零门槛的碎片化推理娱乐体验，摆脱传统海龟汤需要真人主持的限制。

## 1.2 核心价值

- 解决单人海龟汤无主持的痛点，实现随时随地自由解谜

- AI智能适配用户推理节奏，精准回应提问，提升互动趣味性

- 打造神秘悬疑的视觉氛围与仪式感交互，差异化普通文字解谜产品

- 轻量化部署、多端适配，满足用户碎片化休闲娱乐需求

## 1.3 版本规划

- **V1.0 核心版**：落地游戏大厅、游戏页面、汤底页面三大必做功能，完成AI解谜全闭环

- **V2.0 扩展版**：迭代用户登录、游戏记录、排行榜、自定义海龟汤、多人作战等增值功能，强化社交性与用户粘性

# 二、目标用户

## 2.1 核心用户画像

- **年龄区间**：16-35岁，学生、职场年轻群体为主

- **用户特征**：热衷悬疑、推理、猎奇内容，喜欢文字互动类游戏，接受AI新玩法，追求沉浸式体验，习惯碎片化娱乐

- **行为习惯**：日常浏览悬疑故事、玩轻量手游、刷推理类短视频，乐于尝试新鲜互动产品

## 2.2 用户痛点

- 传统海龟汤依赖真人主持，单人无法体验，组队门槛高

- 普通解谜游戏玩法单一，互动性弱，缺乏沉浸感

- 碎片化时间缺少轻量化、高趣味性的推理娱乐选择

# 三、核心功能模块

## 3.1 V1.0 必做核心功能

### 3.1.1 游戏大厅

- **功能定位**：游戏主入口，承载故事展示、筛选、选择功能

- **核心能力**：
        

    1. 采用**卡片式网格布局**，移动端单列、PC端双列展示，视觉整洁有序

    2. 单张卡片包含：故事标题、难度等级（简单/中等/困难）、简短封面引导语

    3. 支持按难度筛选故事，默认按推荐/最新排序

    4. 顶部展示产品LOGO、名称，卡片悬停/点击有轻微上浮反馈

    5. 点击卡片直接跳转对应游戏对局页面

### 3.1.2 游戏页面

- **功能定位**：核心解谜交互页面，用户与AI文字互动，推进推理流程

- **核心能力**：
        

    1. 界面参考**微信聊天风格**，左侧为AI回复气泡，右侧为用户提问气泡，消息排版清晰

    2. **顶部固定信息栏**：全程展示**故事标题、难度、完整汤面（初始谜面）**，方便用户随时查看线索

    3. 底部配置文字输入框，支持发送提问，AI实时回复「是/否/无关紧要/有一定关系」等标准话术

    4. 增设**提示按钮**：用户可获取AI关键线索，限制每日/单局使用次数

    5. 增设**结束游戏按钮**：高亮配色，用户可随时终止对局，一键返回游戏大厅

    6. 增设揭晓汤底按钮：用户主动触发，跳转真相页面

    7. 本地缓存聊天记录，页面刷新不丢失，保障解谜连贯性

### 3.1.3 汤底页面

- **功能定位**：真相揭晓页面，强化解谜仪式感与收尾体验

- **核心能力**：


    1. 进入页面触发悬念动效（迷雾消散、文字逐行渐显），营造揭晓仪式感

    2. 完整展示故事完整真相、关键推理点解析，梳理解谜逻辑

    3. 配置「返回大厅」「再玩一局」快捷按钮，引导用户二次游戏

    4. 延续悬疑视觉风格，文字动效轻量化，不干扰阅读体验

## 3.2 V2.0 后续规划功能（非必做）

- **用户登录系统**：支持手机号验证码、第三方快捷登录，绑定用户身份

- **游戏记录保存**：云端同步已玩故事、解谜进度、聊天记录，支持断点续玩

- **排行榜**：按解谜速度、完成故事数、高难度通关数进行榜单排名

- **自定义海龟汤**：用户自主创作汤底+汤面，提交审核通过后上架共享

- **多人作战模式**：2-6人联机房间，组队推理、实时互动，提升社交属性

# 四、界面设计规范

## 4.1 整体风格要求

- **视觉基调**：神秘、悬疑、暗黑，贴合海龟汤推理氛围

- **主色调**：深蓝色系（主色#0A192F、辅助色#112244、强调色#64FFDA）

- **字体规范**：无衬线字体，文字对比度适配暗色背景，保证可读性

- **动效规范**：轻量化悬疑动效，拒绝花哨视觉，不干扰核心解谜操作

## 4.2 分页面设计细节

### 4.2.1 游戏大厅

- 卡片采用半透明深蓝色渐变背景，边角做圆角处理

- 难度标签用颜色区分：简单（绿色）、中等（黄色）、困难（红色）

- 卡片留白合理，标题醒目，封面文案精简不杂乱

### 4.2.2 游戏页面

- 汤面展示区：半透明卡片底色，字号大于聊天文字，左对齐排版

- 消息气泡：用户气泡（深蓝色高亮）、AI气泡（半透明浅灰）

- 功能按钮：结束游戏按钮用红色/橙色高亮，放置右上角或底部功能区，与其他按钮分区布局，防止误触

- 输入框圆角设计，发送按钮辨识度高，操作便捷

### 4.2.3 汤底页面

- 全屏暗色背景，真相内容居中排版，文字逐行渐显

- 搭配轻微迷雾/星光动效，强化悬疑仪式感

- 操作按钮大尺寸圆角设计，配色醒目，降低操作成本

# 五、技术栈建议

|技术模块|选型方案|选型优势|
|---|---|---|
|前端|React + TypeScript + Tailwind CSS|组件化开发、类型安全、快速搭建悬疑UI，多端响应式适配|
|后端|Node.js + Express|轻量高效，快速搭建API接口，无缝对接前端与AI服务|
|AI服务|DeepSeek / Claude API|大模型智能应答，精准匹配海龟汤提问规则，回复稳定高效|
|部署方案|Vercel|一键部署、免费额度够用、自动CI/CD，支持全栈项目托管|
---

# 六、非功能性要求

- **响应速度**：AI回复延迟≤2秒，页面加载速度≤1.5秒

- **兼容性**：支持移动端、PC端主流浏览器，适配各类屏幕分辨率

- **稳定性**：核心功能无崩溃，聊天记录、对局状态不丢失

- **易用性**：零操作门槛，无需新手引导即可上手，交互逻辑符合用户直觉
